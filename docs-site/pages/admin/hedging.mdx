# Hedging Dashboard - Admin Panel Guide

## What You'll See

### Current Status

The dashboard shows:
- **System Status**: Healthy, Disabled, Disconnected, or High Exposure
- **Connection Status**: Connected to external liquidity or Disconnected
- **Real-time Metrics**: Success rate, profit, exposure

## How Hedging Works

When users place orders on Pariflow, the system can automatically hedge these positions on external liquidity providers. This ensures:

1. **Risk Management**: Platform exposure is minimized
2. **Price Discovery**: Prices reflect broader market sentiment
3. **Liquidity**: Users get better execution on larger orders

## Dashboard Features

### 1. System Health Banner

Shows current status:
- **Healthy** - All systems operational
- **Disabled** - Hedging turned off
- **High Exposure** - Approaching position limits
- **Disconnected** - External connection unavailable

### 2. Main Metrics Cards

**Success Rate**
- Percentage of successful hedges
- Target: >95%

**Net Profit**
- Total profit after fees
- Includes margin percentage

**Unhedged Exposure**
- USD value of positions without hedges
- Alert if approaching limits

**Avg Hedge Time**
- How long hedges take to execute
- Target: <5 seconds

### 3. Profit Breakdown
- Spread captured (revenue)
- External fees (cost)
- Net profit (after fees)

### 4. Configuration Panel
Shows current settings:
- Hedging enabled/disabled
- Minimum spread requirements
- Maximum position sizes
- Exposure limits

### 5. Top Markets
- Markets with most hedge volume
- Profit per market
- Number of hedges

### 6. Recent Failures
Debug panel showing:
- Why hedges failed
- Order details
- Timestamps

## Getting Started

### Step 1: Obtain Access

Contact your platform administrator to:
1. Request external API access credentials
2. Set up required wallet infrastructure
3. Configure environment variables

### Step 2: Enable Hedging

1. Open admin panel
2. Navigate to Hedging section
3. Click "Enable Hedging" button
4. Monitor the dashboard for successful operation

## Monitoring

### Key Metrics to Watch

- **Success Rate**: Should be >95%
- **Net Profit**: Should be positive after fees
- **Unhedged Exposure**: Keep below configured limits
- **Avg Hedge Time**: Should be <5 seconds

### Alerts

The system will alert you if:
- Success rate drops below threshold
- Unhedged exposure exceeds limits
- External connection is lost
- Performance degrades

## Troubleshooting

### "Disconnected" Status

**Causes:**
- Missing or incorrect credentials
- Network connectivity issues
- External API downtime

**Solutions:**
1. Verify all required environment variables are set
2. Restart the server
3. Check external service status

### Low Success Rate

**Causes:**
- Insufficient external liquidity
- Price slippage too high
- Rate limiting

**Solutions:**
1. Check market liquidity
2. Adjust spread settings
3. Enable order splitting for large orders

### High Unhedged Exposure

**Causes:**
- Many failed hedges
- Large user orders
- System disabled

**Solutions:**
1. Enable hedging if disabled
2. Investigate failure reasons
3. Consider manual intervention for large positions

## Best Practices

1. **Start Conservative**: Begin with higher minimum spreads
2. **Monitor Closely**: Watch metrics during initial operation
3. **Adjust Gradually**: Make small configuration changes
4. **Secure Credentials**: Follow security best practices for all secrets
5. **Test First**: Verify configuration before enabling in production
